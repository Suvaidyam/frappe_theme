# AI Query Report Generator — Implementation Plan

## Overview

A native, multi-provider AI-powered Query Report generator integrated into Frappe Theme. Users describe reports in natural language, and AI generates the SQL query, columns, and filters — creating a fully functional Frappe Query Report.

## UX Flows

### Flow 1: Standalone Generator Page
- New Frappe page at `/app/ai-report-generator`
- Chat-style interface: user describes what they want
- AI generates SQL + columns + filters with live preview
- User can iterate ("add a date filter", "group by company")
- "Save as Report" button creates a real Frappe Query Report

### Flow 2: Inline Assist in Report Builder
- "AI Assist" button added to the Query Report form (when editing `query` field)
- Opens a slide-over/dialog with prompt input
- AI generates/modifies the SQL query in-place
- Also suggests columns and filters to add

## Implementation Steps

### Step 1: AI Provider Settings in My Theme

Add new section to `my_theme.json` under the Integrations tab:

**Fields:**
- `ai_provider_section` — Section Break, label "AI Report Generator"
- `ai_provider` — Select: "OpenAI", "Anthropic"
- `ai_api_key` — Password field (encrypted at rest)
- `ai_model` — Data field with default per provider (gpt-4o / claude-sonnet-4-5-20241022)

### Step 2: LLM Provider Abstraction

**`controllers/llm_provider.py`** — Base class:
```python
class LLMProvider:
    def generate(self, system_prompt, user_prompt) -> str

class OpenAIProvider(LLMProvider): ...
class AnthropicProvider(LLMProvider): ...

def get_provider() -> LLMProvider  # Factory from My Theme settings
```

Uses raw HTTP requests (no pip dependencies) to call OpenAI/Anthropic APIs.

### Step 5: AI Prompt Engineering

System prompt structure:
1. Role: "You are a Frappe/MariaDB SQL expert"
2. Schema context: Tables, columns, types, relationships
3. Rules: Use backtick quoting (`tabDocType`), valid Frappe fieldtypes for columns, proper filter format
4. Output format: JSON with {query, columns, filters, explanation}

### Step 6: Standalone Page

**`frappe_theme/page/ai_report_generator/`**:
- `ai_report_generator.json` — Page definition
- `ai_report_generator.js` — Chat UI with:
  - DocType multi-select (scopes schema context)
  - Chat input for natural language prompts
  - SQL preview with syntax highlighting
  - Column/filter preview table
  - "Run Preview" button (executes with LIMIT 20)
  - Data preview table
  - "Save as Report" button
  - Conversation history for iterative refinement

### Step 7: Inline Report AI Assist

**`public/js/report_ai_assist.bundle.js`**:
- Adds "AI Assist" button to Query Report form
- Opens dialog with prompt input
- Generates/replaces the `query` field content
- Suggests columns and filters
- Registered via `doctype_js` for Report DocType

### Step 8: Hooks Registration

In `hooks.py`:
- Add `ai_report_generator` page

- Add new bundle to `app_include_js` if needed

## Security Considerations

- API keys stored as Password field (encrypted by Frappe)
- Generated SQL validated via `check_safe_sql_query()` before execution
- Preview queries run with LIMIT cap and user permission context
- Only users with System Manager or Report Manager role can use the generator
- All queries execute within Frappe's permission framework
